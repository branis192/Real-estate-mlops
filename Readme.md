## 📊 Data Engineering & Preprocessing

To prevent data leakage and ensure reproducibility in production environments, all preprocessing operations are integrated into a Scikit-Learn `ColumnTransformer`.

### Data Cleaning

* Removed extreme outliers where `Gr Liv Area > 4000 sq ft`
* Applied domain-driven preprocessing based on exploratory analysis

### Target Transformation

The target variable (`SalePrice`) was transformed using:

```python
np.log1p(SalePrice)
```

Benefits:

* Reduces skewness
* Stabilizes variance
* Improves linear model performance

### Numerical Features

Processing steps include:

* Missing value imputation

  * Structural features → filled with `0`
  * Other numerical features → median imputation
* Feature scaling using `StandardScaler`

### Categorical Features

Processing steps include:

* Structural missing values → replaced with `"None"`
* Remaining missing values → imputed using the most frequent category
* Encoding using:

```python
OneHotEncoder(handle_unknown="ignore")
```

## 🤖 Model Benchmarking

Models were evaluated using a 20% holdout test set.

### Evaluation Metric

**Mean Absolute Error (MAE)** measured on the original price scale (USD).

| Model                   | Test MAE ($)  | Status               | Comments                                                                           |
| ----------------------- | ------------- | -------------------- | ---------------------------------------------------------------------------------- |
| Linear Regression       | **12,161.84** | 🏆 Selected Baseline | Excellent performance after feature engineering and target transformation          |
| Random Forest Regressor | 14,466.52     | Challenger           | Less effective with high-dimensional sparse features generated by one-hot encoding |

### Key Insight

Despite its simplicity, Linear Regression outperformed Random Forest due to:

* Effective logarithmic target transformation
* Robust preprocessing pipeline
* High-dimensional sparse feature space compatibility

## 🚀 Running the Project

### 1. Clone the Repository

```bash
git clone <your-repository-url>
cd real-estate-mlops
```

### 2. Create a Virtual Environment

```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

### 4. Train the Model

Execute the production training pipeline:

```bash
python3 src/train.py
```

This script will:

1. Load the dataset
2. Split training and testing data
3. Build the preprocessing pipeline
4. Train the model
5. Evaluate performance
6. Export the trained pipeline to:

```bash
models/model_pipeline.joblib
```
